# 跨项目复用：理想之地 → trade-debt（2026-05-23）

## 背景
将 workspace-audit 的全套方法论从理想之地项目复用到金瀚债权（trade-debt）项目。trade-debt的底子更好——源文件/已按01-09编号分类，wiki有12页/三层结构。

## 执行步骤

### Step 1: 诊断
对照workspace-audit的诊断清单检查目标项目，发现4类问题：

| 问题 | 详情 | 严重度 |
|------|------|--------|
| .venv脏数据 | akshare-mcp-server/.venv 7,748个文件/259.9MB | 🔴 最大单项目 |
| 输出散乱 | 仅1个`06-策略方案/`目录，编号不连续 | 🟡 |
| wiki孤立 | 12页中7页无入链、2页无出链 | 🟢 |
| SOUL.md路径 | 引用旧的`06-策略方案\`路径 | 🟢 |

### Step 2: 清理
```
① 删 .venv          → 7,748个文件/259.9MB → 释放
② 输出/重构         → 06-策略方案/ → 01-策略/ + 02-脚本/
③ wiki链接修复      → 0断裂链/0孤出链
④ akshare-mcp-server → 合并到02-脚本/
⑤ 10组重复文件去重   → 12.4MB释放
⑥ 7个空目录删除
```

### Step 3: SOUL.md升级
- 路径更新：`06-策略方案\` → `01-策略\` + `02-脚本\`
- 注入三步分流工作流（适配trade-debt场景）

### Step 4: .obsidian复用
- 复制Landofdream的.obsidian配置到TradeDebt
- Remote Save同步路径独立配置为`tradedebt`

## 可复用的经验
- **输出目录模板**应适配业务：投后管理→策略/脚本/数据；房地产→脚本/舆情/数据
- SOUL.md的三步分流章节具有普适性，只需替换目录名
- .venv清理可自动化——两个项目累计清理~450MB，是所有项目通用的脏数据
- "先诊断再动手+结构化输出"模式在两个项目验证有效
