# 红旗信号手册（制造业/硬科技版）

## 红旗信号严重度分级

| 级别 | 含义 | 典型交易动作 |
|------|------|-------------|
| P0 交易杀手 | 投资逻辑被打破，或存在致命法律/合规缺陷 | 拒绝、暂停，或要求彻底整改后方可上会 |
| P1 重大条件 | 需要在价格、条款、证据、第三方验证上有重大调整 | 重新定价、交割先决条件、赔偿/托管、专业审查 |
| P2 监控项 | 可管理的问题，投后跟踪即可 | 制定整改计划，明确责任人和时间表 |

## 红旗卡片

### 卡片1：收入真实性

**触发条件：**
- 收入增长无订单、交货、验收、发票、物流、回款、客户确认支持。
- 期末集中确认大额收入。

**检查方法：**
- 抽样追踪交易从合同/订单→交货→验收→发票→银行回款→会计入账的全链条。
- 按客户比对收入、应收账款、后续回款、信用期、客户访谈结果。
- 检查期末交易、整数金额合同、同一地址的交易对手、异常关联方。

**交易动作：** P0。暂停正面推荐，直至抽样测试和客户确认为止。无法解决则重新定价或拒绝。

### 卡片2：利润/毛利率注水

**触发条件：**
- 毛利率或EBITDA利润率显著高于同业或历史趋势，且无可记录的商业逻辑。
- 报告利润与经营现金流持续背离。

**检查方法：**
- 按产品/客户/渠道重建毛利率。
- 检查研发费用资本化、劳动力/渠道返利/退货/坏账/存货跌价/补贴/一次性收益。
- 比对产能/物流/用工/原材料消耗与报告销量的关系。

**交易动作：** P1。使用调整后盈利做估值。要求价格调整或earnout保护。

### 卡片3：商业包装

**触发条件：**
- 低壁垒或贸易型业务被包装为高科技/平台/SaaS/深度制造。

**检查方法：**
- 识别实际增值活动：设计、制造、IP、数据、流程、客户所有权，还是仅转售/中介。
- 核查员工岗位分布、研发产出、产品规格、专利、客户合同、供应商依赖度。

**交易动作：** P1。重新归类估值方法和可比公司。若估值依赖包装出来的标签，则重新定价或拒绝。

### 卡片4：关联交易依赖

**触发条件：**
- 与创始人家族、集团兄弟公司、未披露关联方之间存在重大销售、采购、融资、资产、租赁、人员往来。

**检查方法：**
- 按所有权、地址、电话、管理层、银行流水、交易模式、信息披露映射交易对手关系。
- 评估必要性、定价公允性、可替代性、投资/上市后能否独立运行。

**交易动作：** P1。要求披露、清理、定价政策、终止/过户安排、赔偿、治理保留事项。

### 卡片5：现金流/营运资金压力

**触发条件：**
- 收入/利润增长同时，应收账款、存货、合同资产、预付款的增速明显快于现金回款。
- DSO持续延长。

**检查方法：**
- 分析DSO、存货周转天数、合同资产账龄、后续回款、坏账、信用政策。
- 测试增长是否需要隐藏融资（来自供应商、银行、股东贷款）。

**交易动作：** P1。根据营运资金需求调整估值；增加营运资金pegging、财务承诺条款或融资条件。

### 卡片6：客户/供应商集中度

**触发条件：**
- 少量客户或供应商驱动了绝大部分收入、毛利、技术获取或生产连续性。
- 单一客户占比超50%。

**检查方法：**
- 核查合同期限、续约、定价、排他性、切换成本、控制权变更同意条款、客户推荐信。
- 供应商侧测试替代品、交货期、价格传导能力、制裁/出口管控风险。

**交易动作：** P1。要求客户/供应商确认函、留存计划、集中度估值折扣、或合同续约为交割先决条件。

### 卡片7：IP/技术所有权

**触发条件：**
- 核心技术由创始人个人、高校、医院、前雇主、关联公司或许可方所有，而非目标公司。

**检查方法：**
- 核查专利/软件著作权/商标/许可证的链条、发明人雇佣关系、权利归属、负担、侵权纠纷。
- 确认目标能否使用、转让、再许可、商业化并抗辩该等IP。

**交易动作：** P0/P1。要求交割前完成权利归属/许可清理、IP陈述保证、专家意见、交割先决条件。

### 卡片8：重要合同控制权变更条款

**触发条件：**
- 关键客户、供应商、融资方、许可方、租赁方、政府合同中包含投资/收购后可终止或需同意的条款。

**检查方法：**
- 核查合同中关于转让、控制权变更、排他性、终止、罚则、价格调整、审计的条款。

**交易动作：** P1。将合同对方同意函设为交割先决条件；若同意时间或结果不确定则做价格折让。

### 卡片9：税务/社保/劳动用工

**触发条件：**
- 少缴税款、非正式雇佣、社保/公积金欠缴、劳务派遣超比例、未付加班费、创始人个人账户代付。

**检查方法：**
- 核查工资表、用工人数、社保缴纳基数、个税、增值税/所得税申报、发票、当地政策合规性。

**交易动作：** P1。量化敞口；要求专业审查、托管/特别赔偿、或交割前整改。

### 卡片10：环境/安全/行业许可

**触发条件：**
- 许可证缺失、过往处罚记录、安全生产不达标、污染物排放问题、危险品处理、许可范围与实际不符。

**检查方法：**
- 核查许可证、检查记录、处罚记录、整改报告、第三方检测、环评报告、安全生产档案。

**交易动作：** P0/P1。取决于停产风险。要求专业审查和整改作为交割先决条件。

### 卡片11：法律诉讼/担保/隐性负债

**触发条件：**
- 诉讼、仲裁、行政处罚、对外担保、股权质押、股东借款、或有付款、表外义务。

**检查方法：**
- 检索公开数据库、银行函证、征信报告、合同、董事会决议、关联方确认、法律顾问备忘录。

**交易动作：** P0/P1。特别赔偿、托管、作为负债调整、要求解决作为交割先决条件，或若威胁偿债能力/控制权则拒绝。

### 卡片12：管理层诚信

**触发条件：**
- 陈述前后矛盾、隐瞒文件、异常关联交易、激进会计处理、声誉问题、过往纠纷记录。

**检查方法：**
- 交叉核对面谈陈述与文件、第三方源、客户、供应商、员工、历史记录。

**交易动作：** P0。诚信问题可上升为交易杀手，即使财务影响暂时难以量化。

## 报告规则

1. 每个P0/P1红旗必须附交易影响（deal implication），不能只描述情况。
2. 不写"风险可控"——除非明确说了什么控制手段来控制。
3. 不写通用性风险——必须与标的公司的实际暴露挂钩。
4. 区分增长风险和存亡/合规风险：
   - 增长风险改变估值和回报预期
   - 存亡/合规风险可以中止或重构交易
