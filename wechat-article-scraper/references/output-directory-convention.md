# Hermes 成果文件输出目录规范

## 输出根目录

```
E:\Hermes\                    # Windows路径（WSL映射：/mnt/e/Hermes/）
```

## 子目录结构

```
E:\Hermes\
├── 01-报告/            # 正式汇报文档
│   ├── *.docx          # Word排版报告（会议纪要、工作方案、竞品对比）
│   ├── *.html          # HTML展示页面（方案落地页、分析报告可视化版）
│   └── *.md            # Markdown报告/分析文档
│
├── 02-数据分析/        # 数据分析产物
│   ├── 交叉分析报告
│   ├── 数据摘要JSON
│   └── 可视化图表
│
├── 03-竞品分析/        # 竞品研究成果
│   ├── SWOT分析
│   ├── 竞品对比报告
│   └── 竞品信息汇总
│
├── 04-策划方案/        # 营销/活动方案
│   ├── 时尚季方案
│   ├── 营销策略
│   └── 活动策划
│
├── 05-参考素材/        # 学习/参考资料
│   ├── 公众号文章归档
│   └── 行业研究素材
│
├── 06-脚本/            # 可复用脚本工具
│   └── *.py / *.sh
│
├── 07-截图/            # 浏览器截图/页面快照
│
└── 目录说明.md         # 索引文件
```

## 命令

```bash
# 写文件时使用WSL路径
ROOT="/mnt/e/Hermes"
```

## 命名规范

- 格式：`类别-描述.扩展名`（如 `竞品SWOT分析.html`）
- 同一类型多次输出以版本号区分（V1.0 / V2.0）
- 中文文件名清晰表达内容，不用编号

## 清理规则

- WSL `~` 目录：不保留正式文件，仅存运行期间的临时工具脚本，每日凌晨自动清理
- `E:\Hermes\`：正式成果永久保留
- 临时脚本（search*.py / extract*.py / analyze*.py / check*.py）写入 `~` 即可，不归入 Hermes 目录
